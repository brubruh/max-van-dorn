Raspberry Pi Home Server

Status: In Progress (Always)  •  Stack: Raspberry Pi • Linux • TimescaleDB • Grafana • Pi-hole • Networking

Overview

A self-hosted Raspberry Pi server I built and configured entirely on my own. What started as a personal home lab has grown into production infrastructure — the server currently ingests live sensor data from a Kiau Tech environmental monitoring node deployed in New Mexico, on a 10-minute increment cycle.

One of the things I'm most proud of: it came online with almost no problems and was up and running remarkably fast for the complexity involved.

What's Running

Pi-hole

Network-wide DNS-based ad blocker running at the router level. Filters ads and trackers for every device on the network without touching individual devices.

TimescaleDB

A PostgreSQL extension optimized for time-series data. Used to store the continuous stream of sensor readings coming in from Kiau Tech field nodes. Port-forwarded and secured for external connections.

Custom Data Ingestion API

A lightweight API layer that accepts packets from remote sensor nodes, validates and cleans the incoming data, strips any malformed readings, and writes clean records to TimescaleDB. Designed to handle the 10-minute ingestion cycle reliably.

Grafana Dashboard

A live Grafana dashboard connected to TimescaleDB, visualizing the incoming sensor data in real time. Makes it easy to monitor trends, spot anomalies, and present data to Kiau Tech stakeholders.

Live Data: Kiau Tech New Mexico Node

The server is currently ingesting environmental data from a sensor node placed in New Mexico. Data arrives every 10 minutes, is cleaned by the ingestion API, stored in TimescaleDB, and rendered in Grafana. This is real, production data supporting Kiau Tech's regenerative agriculture research.
